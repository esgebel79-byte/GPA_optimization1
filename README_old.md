# Система оптимизации режимов газоперекачивающих агрегатов (ГПА)

## 📋 Назначение
Комплексное приложение для почасового моделирования и многоалгоритмной оптимизации режимов работы группы газоперекачивающих агрегатов. Система поддерживает **четыре метаэвристических алгоритма** для решения задачи оптимизации:

1. **Точное обеспечение** суммарного коммерческого расхода (параметр `Q_target`)
2. **Минимизация** расхода топливного газа и дисбаланса нагрузки между агрегатами
3. **Соблюдение** всех технологических ограничений (диапазоны оборотов, расходов, температур)
4. **Сравнительный анализ** производительности всех алгоритмов с визуализацией

## 📁 Структура файлов

### Core Engine
| Файл | Назначение |
|------|------------|
| `exec.py` | **Главный скрипт** (1461 строк). Организует 24-часовую цикличную симуляцию, управляет режимами регулирования, запускает сравнительный анализ 4 алгоритмов, генерирует визуализации (таблицы, гистограммы, тепловые карты). |
| `RegBase.py` | Абстрактный базовый класс `NstRegulatorBase`. Унифицированный интерфейс для всех регуляторов через метод `calculate(Pin, Tin, Q_target, gpa_list, comp_list, **kwargs)`. |

### Алгоритмы оптимизации (Regulators)
| Файл | Алгоритм | Особенности |
|------|----------|------------|
| `Nst_SHOPreg.py` | **PID** (Proportional-Integral-Derivative) | Классический регулятор: корректирует обороты пропорционально ошибке расхода. Anti-windup защита. Детерминированный. |
| `Nst_PSOreg.py` | **PSO** (Particle Swarm Optimization) | Роевая оптимизация через библиотеку `pyswarms`. 30 частиц × 40 итераций. Стохастический, требует фиксирования seed для воспроизводимости. |
| `Nst_GWOreg.py` | **GWO** (Grey Wolf Optimizer) | Имитирует иерархию охоты серых волков (α/β/δ). 30 волков × 40 итераций. Отслеживает историю сходимости. |
| `Nst_ACOreg.py` | **ACO** (Ant Colony Optimization) | Коллективный интеллект муравьёв с феромонной коммуникацией. 30 муравьёв × 40 итераций. Отслеживает историю феромонов и привлекательности для анализа. |

### Модели физических систем
| Файл | Назначение |
|------|------------|
| `gpa_model.py` | Модель ГПА (`dataclass GPA`). Методы: расчёт расхода, степени сжатия, температуры нагнетания, мощности, расхода топлива, проверка ограничений. |
| `compressor_model.py` | Математическая модель центробежного компрессора (`Compressor`). Аппроксимации для Π(Nst), η(Nst), Q_work(Nst), Q_surge(Nst) без паспортных карт. Блокировка помпажа. |
| `pressure_imit.py` | Генератор суточного профиля входного давления. Режимы: `'daily_cycle'` (пик днём, спад ночью), `'constant'`, `'ramp'`. |
| `gpa_constants.py` | Справочник констант: термодинамические параметры газа, молярная масса, базовые КПД, номинальные параметры ГТУ. |

### Оптимизация и анализ
| Файл | Назначение |
|------|------------|
| `functional.py` | Целевая функция `objective_function`. Объединяет: штраф за отклонение от Q_target, расход топлива, дисбаланс нагрузки, штрафы за нарушение ограничений. |
| `visualize_landscape.py` | Утилита визуализации ландшафта оптимизации (3D-графики поверхностей). |


## 🔄 Архитектура и принцип работы

### Вычислительный цикл: Одна симуляция (24 часа)
```
Час 0-23:
  ├─ Получить Pin(t) из профиля давлений
  ├─ Вызвать regulator.calculate(Pin, Tin, Q_target, gpa_list, comp_list)
  ├─ Регулятор оптимизирует [Nst1, Nst2, Nst3]
  ├─ Обновить состояние компрессоров & ГПА
  ├─ Вычислить objective_function → затраты за час
  └─ Накопить результаты

ИТОГ: Суммарные затраты за сутки + Финальные параметры каждого ГПА
```

### Сравнительный анализ: 4 алгоритма
```
1. Основная симуляция в выбранном режиме (REGULATION_MODE)
   ↓
2. Запуск PSO, GWO, ACO, PID в отдельных, чистых инстансах
   ├─ Каждому алгоритму выделяются свежие объекты GPA/Compressor
   ├─ Все получают одинаковый Pin(t) профиль
   ├─ Каждый работает на одинаковых начальных условиях
   ↓
3. Сбор результатов: Nst, Q, затраты для каждого
   ↓
4. Генерация визуализаций:
   ├─ Таблицы финальных параметров ГПА
   ├─ Графики сходимости (для PSO, GWO, ACO)
   ├─ Гистограммы сравнения (Nst по каждому ГПА, Q суммарный)
   ├─ Сводная таблица результатов
   └─ ACO специфические анализы (феромоны, привлекательность)
   ↓
5. Экспорт в PNG и CSV для отчётности
```

## ⚙️ Конфигурация и запуск

### Основные параметры (в `exec.py`):
```python
REGULATION_MODE = "ACO"      # "PID" | "PSO" | "GWO" | "ACO"
Q_TARGET_DAY = 2400.0        # целевой расход, тыс.м³/сутки
SIMULATION_HOURS = 24        # количество часов в сценарии
Tin_ext = 22.0               # температура на входе, °C

# Профиль входного давления: пик днём, спад ночью
pin_profile = generate_pin_profile(
    hours=24,
    mode="daily_cycle",        # варианты: "daily_cycle", "constant", "ramp"
    base_pin=5.8,              # базовое давление, МПа
    amplitude=0.6,             # амплитуда колебаний, МПа
    noise_std=0.04,            # шум, МПа
    seed=42                    # для воспроизводимости
)
```

### Запуск:
```bash
python exec.py
```
Автоматически:
1. Запустит основную 24-часовую симуляцию в выбранном режиме
2. Запустит сравнительные симуляции для всех 4 алгоритмов
3. Создаст таблицы, графики, тепловые карты
4. Экспортирует результаты в `results/` (PNG и CSV)

## 🔬 Анализ алгоритмов

### Сравнительные метрики (в итоговой таблице):
- **Затраты (Functional Cost)** — чем меньше, тем лучше
- **Точность достижения Q** — ошибка в тыс.м³, ближе к 0 лучше
- **Сбалансированность** — отношение max/min объёмов между ГПА
- **Стабильность** — вариативность оборотов по часам
- **Среднее Nst** — средние обороты по сутке
- **Среднее Q** — средний расход за сутку

### Визуальная интерпретация:
- **Сетка гистограмм** показывает распределение оборотов каждого ГПА для каждого алгоритма
- **Группировка по цветам** (синий=PSO, оранжевый=GWO, зелёный=ACO, красный=PID)
- **Высота столбца** = среднее значение, **погрешность** = std dev

### ACO анализ (специфичный):
- **Heatmap феромонов**: Где конкретно алгоритм "запомнил" хорошие решения (красные области = конвергированные решения)
- **График влияния**: Какой компонент доминировал на каждой итерации
  - Ранние итерации: высокая привлекательность → **исследование (exploration)**
  - Поздние итерации: высокие феромоны → **эксплуатация (exploitation)**
- `bounds` оптимизатора `pyswarms` ограничивает пространство поиска.
- В `Nst_PSOreg.py` применяется `np.clip()` перед передачей оборотов в модель и на выходе из `calculate()`, что гарантирует строгое соблюдение технологических лимитов.

## 📝 Примечания
- Модель компрессора (`compressor_model.py`) использует аппроксимационные зависимости без паспортных карт, что указано в документации модуля.
- PSO-регулятор содержит стохастическую составляющую; для полной воспроизводимости результатов рекомендуется фиксировать seed в `numpy` и `pyswarms` перед запуском.
- Все расчёты выполняют приведение давлений к абсолютным и температур к Кельвинам в соответствии с константами из `gpa_constants.py`.
