# Система оптимизации режимов газоперекачивающих агрегатов (ГПА)

## 📋 Назначение
Программный комплекс выполняет почасовое моделирование и оптимизацию режимов работы группы газоперекачивающих агрегатов. Решение оптимизационной задачи направлено на достижение двух целей:
1. Точное обеспечение суммарного коммерческого расхода, заданного параметром `Q_target`.
2. Минимизация суммарного расхода топливного газа и сбалансированное распределение нагрузки между агрегатами при соблюдении технологических ограничений.

## 📁 Структура файлов

| Файл | Назначение |
|------|------------|
| `exec.py` | Главный скрипт симуляции. Инициализирует объекты ГПА и компрессоров, выбирает стратегию регулирования, организует почасовой цикл расчёта, выводит протокол и аккумулирует значение функционала. |
| `Nst_PSOreg.py` | Регулятор на основе метода роя частиц (GlobalBestPSO, `pyswarms`). Выполняет многомерную оптимизацию оборотов всех ГПА одновременно, жёстко ограничивая поиск границами `Nst_min_lim` / `Nst_max_lim`. |
| `SHOPreg_base.py` | ПИД-регулятор (`PIDRegulator`). Корректирует обороты на основе ошибки по суммарному расходу, использует anti-windup для интегральной составляющей и распределяет корректировку пропорционально гибкости рабочих диапазонов агрегатов. |
| `Nst_SHOPreg.py` | Абстрактный базовый класс `NstRegulatorBase`. Определяет интерфейс метода `calculate()`, обеспечивая полиморфизм стратегий регулирования. |
| `functional.py` | Целевая функция `objective_function`. Формирует скалярную оценку режима: штраф за отклонение от `Q_target`, расход топлива, дисбаланс нагрузки и суммы штрафов за нарушение ограничений по расходу и оборотам. |
| `gpa_model.py` | Модель газоперекачивающего агрегата (`dataclass GPA`). Содержит методы расчёта коммерческого расхода, степени сжатия, температур нагнетания, эффективной мощности, расхода топлива и проверки технологических лимитов. |
| `compressor_model.py` | Упрощённая математическая модель центробежного нагнетателя (`Compressor`). Аппроксимирует зависимости степени сжатия, КПД, рабочего расхода и расхода помпажа от частоты вращения без использования паспортных характеристик. |
| `gpa_constants.py` | Модуль неизменяемых физических и эксплуатационных констант: термодинамические базовые условия, свойства газа, молярная масса, коэффициенты КПД, номинальные параметры ГТУ. |

## ⚙️ Принцип работы системы

1. **Инициализация:** В `exec.py` создаются списки объектов `GPA` и `Compressor` с заданными паспортными ограничениями.
2. **Выбор стратегии:** Переменная `REGULATION_MODE` (`"PID"` или `"PSO"`) определяет используемый регулятор. Оба наследуют `NstRegulatorBase` и реализуют метод `calculate()`.
3. **Расчёт режима:** На каждом шаге регулятор принимает внешние условия (`Pin`, `Tin`, `Q_target`) и возвращает вектор рекомендуемых оборотов `[Nst_1, Nst_2, Nst_3]`.
4. **Обновление состояния:** Оборты передаются в модели компрессоров (`comp.get_state()`), после чего последовательно вызываются методы `gpa.update_state()`, `calc_production_flow()`, `calc_compression_power()`, `calc_available_power()`, `calc_fuel_flow()`.
5. **Оценка функционала:** Результаты передаются в `objective_function`, где вычисляется суммарная стоимость режима. Значение аккумулируется для итоговой отчётности.

## 🚀 Запуск и настройка
### Запуск симуляции
```bash
python exec.py

### Настройка параметров PSO
В `exec.py` при инициализации `PyswarmsRegulator`:
```python
regulator = PyswarmsRegulator(n_particles=30, iters=40, c1=1.49445, c2=1.49445, w=0.7298)
```

### Настройка весов функционала
В `functional.py` словарь `weights`:
```python
weights = {"w_Q": 1000.0, "w_F": 1.0, "w_B": 10.0}
```
- `w_Q` — приоритет достижения целевого расхода.
- `w_F` — вес расхода топливного газа.
- `w_B` — вес дисбаланса нагрузки между ГПА.

### Ограничение оборотов
Условие `Nst = min(Nst, Nst_max)` (и аналогично для минимума) реализовано на двух уровнях:
- `bounds` оптимизатора `pyswarms` ограничивает пространство поиска.
- В `Nst_PSOreg.py` применяется `np.clip()` перед передачей оборотов в модель и на выходе из `calculate()`, что гарантирует строгое соблюдение технологических лимитов.

## 📋 Технические требования
- Python 3.8+
- `numpy`
- `pyswarms`
- Стандартная библиотека: `typing`, `abc`, `dataclasses`, `math`, `time`

## 📝 Примечания
- Модель компрессора (`compressor_model.py`) использует аппроксимационные зависимости без паспортных карт, что указано в документации модуля.
- PSO-регулятор содержит стохастическую составляющую; для полной воспроизводимости результатов рекомендуется фиксировать seed в `numpy` и `pyswarms` перед запуском.
- Все расчёты выполняют приведение давлений к абсолютным и температур к Кельвинам в соответствии с константами из `gpa_constants.py`.
